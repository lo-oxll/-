# نظام السيطرة المخزنية والمحاسبية

نظام احترافي متكامل لإدارة المخازن والمحاسبة، مبني بـ **HTML / CSS / JavaScript (Vanilla)** من الواجهة، ومتصل بقاعدة بيانات **Supabase** (PostgreSQL + مصادقة + صلاحيات على مستوى الصف).

## المزايا الرئيسية

- **استلام مخزني**: تسجيل الوارد بسعر الوصل الفعلي لكل مادة.
- **إصدار مخزني**: السعر يُحسب تلقائياً بـ **المتوسط الوزني المرجّح (Weighted Average)** لحظة الحفظ — منطق محسوب في قاعدة البيانات نفسها (Trigger)، وليس فقط بالواجهة، لضمان الدقة.
- **قيود محاسبية تلقائية بنظام القيد المزدوج (Debit/Credit)**: كل وثيقة استلام أو إصدار تُنشئ قيداً محاسبياً متوازناً تلقائياً (مدين/دائن) مرتبطاً بدليل الحسابات.
- **قيود يدوية**: لإدخال أي حركة محاسبية أخرى، مع تحقق فوري من توازن القيد قبل الترحيل.
- **تقارير مالية**: ميزان المراجعة، الميزانية العمومية، قائمة الدخل (الأرباح والخسائر) — مُستخلَصة مباشرة من القيود.
- **صلاحيات مستخدمين متعددة**: مدير نظام (admin) / محاسب (accountant) / مدير (manager) / مدقق (auditor)، مطبّقة بـ Row Level Security على مستوى قاعدة البيانات نفسها (وليس فقط بإخفاء أزرار بالواجهة).
- **تنبيهات حد أدنى للمخزون وإعادة الطلب** مع شريط نسبة مرئي.
- **تصدير/طباعة تقارير رسمية**: وثائق الاستلام والإصدار، وكل تقرير مالي، تُطبع بترويسة رسمية جاهزة كـ PDF عبر مربع طباعة المتصفح (يدعم العربية بشكل كامل بدون مشاكل تشكيل الحروف، بعكس مكتبات PDF الجاهزة).
- **سجل مراجعة (Audit Log)** غير قابل للتعديل: يسجل كل عملية حساسة (تسجيل دخول، إنشاء وثيقة، تغيير دور مستخدم...).
- **بحث فوري بالذكاء الذاتي (Autocomplete)** عن المواد بالاسم أو الرقم المخزني عند إدخال أي وثيقة.

## خطوات الإعداد

### 1) إنشاء مشروع Supabase
1. افتح [supabase.com](https://supabase.com) وأنشئ مشروعاً جديداً مجانياً.
2. من القائمة الجانبية → **SQL Editor** → **New query**، الصق محتوى الملف `sql/schema.sql` كاملاً واضغط **Run**.
   - هذا ينشئ كل الجداول، العلاقات، التريغرات (حساب السعر الوسطي + القيود التلقائية)، السياسات الأمنية (RLS)، ودليل حسابات أولي بسيط.

### 2) ربط الواجهة بالمشروع
1. من **Project Settings → API** انسخ:
   - `Project URL`
   - `anon public key`
2. افتحي/افتح `js/config.js` وضع القيمتين بدل `YOUR-PROJECT` و`YOUR-ANON-PUBLIC-KEY`.

### 3) إنشاء أول مستخدم (مدير نظام)
1. افتح `index.html` (يمكن فتحه مباشرة بالمتصفح، أو رفعه لأي استضافة ثابتة مثل GitHub Pages).
2. اضغط **إنشاء حساب جديد** وسجّل بياناتك — سيُنشأ الحساب تلقائياً بدور "محاسب".
3. ارجع إلى Supabase SQL Editor ونفّذ:
   ```sql
   update profiles set role='admin' where full_name='اسمك الكامل كما أدخلته';
   ```
4. سجّل الدخول من جديد — ستظهر لك كل صفحات الإدارة (المخازن، المستخدمون، دليل الحسابات...).

### 4) البيانات الأولية المطلوبة قبل البدء
- **المخازن** (صفحة "المخازن"، دور admin): أضف مخزناً واحداً على الأقل.
- **دليل المواد** (صفحة "دليل المواد"): أضف المواد التي تتعامل بها (الرقم المخزني، الاسم، الوحدة، حد إعادة الطلب).
- دليل الحسابات جاهز بحسابات أساسية (المخزون، الموردون، مصروف المواد المصروفة...)؛ يمكن للـ admin إضافة حسابات أخرى من صفحة "دليل الحسابات".

## هيكلية المشروع

```
index.html              الهيكل العام + شاشة الدخول
css/styles.css          كل الأنماط (توكنات تصميم + وضع فاتح/داكن + أنماط الطباعة)
js/config.js            بيانات الاتصال بـ Supabase
js/supabaseClient.js    طبقة الوصول للبيانات (Data Access Layer) — كل استعلامات القاعدة من هنا
js/app.js               التوجيه بين الصفحات + أدوات مشتركة (toast, autocomplete, الشريط الجانبي)
js/auth.js              تسجيل الدخول / حساب جديد / خروج
js/inventory.js         المخازن، دليل المواد، الاستلام، الإصدار، الأرصدة، تنبيهات الحد الأدنى
js/accounting.js        دليل الحسابات + القيود المحاسبية (تلقائية ويدوية)
js/reports.js           التقارير المالية + منطق الطباعة الرسمية
js/admin.js             إدارة المستخدمين والصلاحيات + سجل المراجعة
sql/schema.sql          مخطط قاعدة البيانات الكامل (جداول + تريغرات + RLS + Views)
```

## ملاحظات مهمة حول منطق التسعير

- **الاستلام**: السعر يُدخل يدوياً من وصل المورّد لكل مادة. عند الحفظ، تريغر `fn_receipt_item_apply` يعيد حساب **السعر الوسطي المرجّح** الجديد للمادة في ذلك المخزن:
  `السعر الوسطي الجديد = ((الرصيد القديم × السعر القديم) + (الكمية الواردة × سعر الوصل)) ÷ (الرصيد القديم + الكمية الواردة)`
- **الإصدار**: السعر لا يُدخل يدوياً؛ تريغر `fn_issue_item_apply` يعبّئه تلقائياً من السعر الوسطي الحالي وينقص الرصيد فقط (لا يُغيَّر السعر الوسطي بعملية الإصدار، فقط الاستلام يغيّره).
- الرصيد **لا يمكن أن يصبح سالباً**: أي محاولة إصدار كمية أكبر من المتاح تُرفض من قاعدة البيانات نفسها.

## اقتراحات للتوسّع لاحقاً
- ربط طلبات الشراء (Purchase Requests) بوثائق الاستلام تلقائياً.
- تصدير Excel لكل الجداول (عبر مكتبة SheetJS، كما في الملف السابق الذي بنيته).
- تقارير حسب المخزن/المادة عبر فترة زمنية محددة (فلترة تاريخ).
- إشعارات بريد إلكتروني عند وصول مادة للحد الأدنى (عبر Supabase Edge Functions).
